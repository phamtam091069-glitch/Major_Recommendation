### 3.2.1.3 Xử lý giá trị ngoại lệ (Outliers)

**Phương pháp 1: IQR (Interquartile Range) - dùng trong project**

```python
# File: data/clean_data.py (dòng 147-172)
Q1 = df[col].quantile(0.25)  # 25th percentile
Q3 = df[col].quantile(0.75)  # 75th percentile
IQR = Q3 - Q1
lower_bound = Q1 - 1.5 * IQR  # Giới hạn dưới
upper_bound = Q3 + 1.5 * IQR  # Giới hạn trên
outlier_mask = (df[col] < lower_bound) | (df[col] > upper_bound)
```

**Phương pháp 2: Z-score**

```python
# File: data/clean_data.py (dòng 174-200)
z_scores = np.abs((df[col] - mean) / std)
outlier_mask = z_scores > 3.0  # Outlier nếu z-score > 3
```

**Xử lý outliers - Winsorization (Cap):**

```python
# Thay thế outlier bằng giới hạn thay vì xóa
df[col] = df[col].clip(lower=lower_bound, upper=upper_bound)
```

Cách này giữ tất cả các hàng, chỉ giới hạn giá trị cực đoan.

**Phương pháp 3: Isolation Forest (Advanced)**

```python
# File: data/clean_data.py (dòng 15, import)
from sklearn.ensemble import IsolationForest

# Phát hiện outlier bằng Isolation Forest
iso_forest = IsolationForest(
    contamination=0.05,  # Giả sử 5% dữ liệu là outlier
    random_state=42
)

outlier_predictions = iso_forest.fit_predict(df_numeric)
# -1 = outlier, 1 = normal

outlier_mask = outlier_predictions == -1
outlier_indices = df[outlier_mask].index.tolist()
```

**So sánh 3 phương pháp:**

| Phương pháp | Ưu điểm | Nhược điểm |
|------------|---------|-----------|
| **IQR** | Đơn giản, nhanh | Chỉ hoạt động tốt trên dữ liệu 1D |
| **Z-score** | Xác định rõ ràng | Yêu cầu dữ liệu bình thường |
| **Isolation Forest** | Phát hiện anomaly phức tạp | Tốn tài nguyên hơn |

**Khi nào dùng phương pháp nào:**
- **IQR**: Dữ liệu numeric đơn giản (kích thước dataset nhỏ)
- **Z-score**: Dữ liệu gần bình thường
- **Isolation Forest**: Dữ liệu đa chiều, anomaly phức tạp

## 3.2.2 CHUẨN HÓA VĂN BẢN TIẾNG VIỆT (TEXT NORMALIZATION)

### 3.2.2.1 Tại sao cần chuẩn hóa tiếng Việt?

Tiếng Việt có nhiều đặc điểm riêng:
- **Dấu thanh**: Dấu sắc, huyền, hỏi, ngã, nặng làm thay đổi ý nghĩa từ
  - Ví dụ: "Kỹ thuật" vs "ky thuat" (không dấu)
- **Khoảng trắng thừa**: " Kỹ Thuật " cần được trim
- **Chữ hoa/thường**: "KỲ THUẬT" vs "kỹ thuật"
- **Ký tự đặc biệt**: "&", "-" cần được xử lý

**Mục đích chuẩn hóa:**
- Để dữ liệu input từ người dùng khớp với dữ liệu huấn luyện
- Tránh mô hình nhầm lẫn do các biến thể của cùng một từ

### 3.2.2.2 Quy trình chuẩn hóa trong project

**File: utils/features.py - Hàm row_dict_from_payload()**

```python
# Chuẩn hóa từng trường categorical
def row_dict_from_payload(payload):
    normalized = {}
    for field, value in payload.items():
        if field in CATEGORICAL_COLS:
            # Bước 1: Chuyển thành chuỗi
            v = str(value).strip()
            
            # Bước 2: Loại bỏ dấu (unidecode)
            v_no_diacritic = unidecode(v)
            
            # Bước 3: Chuyển thành chữ thường
            v_lower = v_no_diacritic.lower()
            
            # Bước 4: Xóa khoảng trắng thừa
            v_final = v_lower.strip()
            
            normalized[field] = v_final
    return normalized
```

**Các bước chuẩn hóa chi tiết:**

1. **Loại bỏ dấu (Diacritical Removal):**
   ```
   Input:  "Kỹ Thuật"
   Output: "Ky Thuat"
   ```
   Sử dụng thư viện `unidecode` hoặc `unicodedata`

2. **Chuyển thành chữ thường (Lowercase):**
   ```
   Input:  "Ky Thuat"
   Output: "ky thuat"
   ```

3. **Xóa khoảng trắng thừa (Strip & Normalize Spaces):**
   ```
   Input:  "  ky  thuat  "
   Output: "ky thuat"
   ```

## 3.2.4 MÃ HÓA DỰ LIỆU PHÂN LOẠI (ONE-HOT ENCODING)

### 3.2.4.1 Tại sao cần One-Hot Encoding?

Mô hình Machine Learning (Random Forest, Logistic Regression) chỉ hiểu dữ liệu số (numeric). Dữ liệu categorical cần được chuyển đổi:

```
Trước: so_thich_chinh = "cong nghe"
Sau:   [1, 0, 0, 0, 0, 0, 0]  (vị trí 0 = "cong nghe")

Trước: so_thich_chinh = "kinh doanh"
Sau:   [0, 1, 0, 0, 0, 0, 0]  (vị trí 1 = "kinh doanh")
```

### 3.2.4.2 One-Hot Encoding trong project

**File: train_model.py (dòng 50-55)**

```python
from sklearn.preprocessing import OneHotEncoder
from scipy.sparse import hstack

def build_feature_matrix(df, encoder, tfidf):
    # Chuẩn hóa categorical columns
    x_cat = df[CATEGORICAL_COLS].fillna("khong xac dinh")
    x_cat = x_cat.apply(lambda col: col.astype(str).str.strip().str.lower())
    
    # One-Hot Encoding
    x_cat_encoded = encoder.transform(x_cat)  # Shape: (n_samples, n_features)
    
    # Kết hợp với TF-IDF (sẽ nói ở phần tiếp)
    x_text = tfidf.transform(df["profile_text"])
    return hstack([x_cat_encoded, x_text])
```

## 3.2.5 XỬ LÝ NLP CHO CÂU TỰ LUẬN (TF-IDF VECTORIZER)

### 3.2.5.1 Giới thiệu TF-IDF

TF-IDF (Term Frequency - Inverse Document Frequency) chuyển đổi văn bản thành vector số:
- **TF**: Tần suất của từ trong văn bản
- **IDF**: Độ hiếm của từ trong tất cả văn bản
- **Mục đích**: Các từ quan trọng (hiếm nhưng xuất hiện trong văn bản) được trọng số cao

**Ví dụ:**
```
Văn bản 1: "Em rất quan tâm đến lập trình phần mềm web API"
Văn bản 2: "Em yêu thích lập trình ứng dụng di động"

TF-IDF Vector:
- "lập": tần suất cao, IDF trung bình → trọng số trung bình
- "em": tần suất cao, IDF thấp (xuất hiện ở cả hai) → trọng số thấp
- "API": tần suất thấp, IDF cao (hiếm) → trọng số cao ✓
```

## 3.2.6 TẠO DỮ LIỆU NHÂN TẠO & CÂN BẰNG (SYNTHETIC DATA & BALANCING)

### 3.2.6.1 Tại sao cần tạo dữ liệu nhân tạo?

Dữ liệu thực tế thường:
- Không đủ mẫu cho tất cả lớp
- Mất cân bằng (một lớp nhiều mẫu, lớp khác ít mẫu)
- Có outliers, noise làm model khó học

**Giải pháp:**
- Tạo dữ liệu synthetic (nhân tạo) cân bằng cho tất cả lớp
- Thêm controlled noise để model học được robustness
- Tạo hard negatives để model phân biệt tốt hơn

### 3.2.6.2 Dữ liệu nhân tạo trong project

**File: data/generate_balanced_students.py**

**1. Tạo dữ liệu cân bằng:**
```python
ROWS_PER_MAJOR = 150  # Mỗi ngành 150 mẫu
HOLDOUT_ROWS_PER_MAJOR = 50  # Test set: 50/ngành

# Kết quả:
# Train: 150 × 67 ngành = 10,050 mẫu
# Holdout: 50 × 67 ngành = 3,350 mẫu (không noise, sạch)
```

**2. Thêm controlled noise:**
```python
NOISE_RATE = 0.45  # 45% mẫu bị thêm noise

def inject_controlled_noise(row, rng):
    if rng.random() >= NOISE_RATE:
        return row  # 55% không thay đổi
    
    # 45%: Ngẫu nhiên thay đổi 1-2 trường
    n_fields = 1 if rng.random() < 0.8 else 2
    fields = rng.sample(list(FIELD_POOLS.keys()), k=n_fields)
    for field in fields:
        candidates = [x for x in FIELD_POOLS[field] if x != row[field]]
        if candidates:
            row[field] = rng.choice(candidates)
    return row
```

**3. Tạo hard negatives:**
```python
HARD_NEGATIVE_RATE = 0.50  # 50% mẫu nhận hard negative

CONFUSION_PAIRS = {
    "Cong nghe thong tin": "He thong thong tin quan ly",
    "Khoa hoc du lieu": "Cong nghe thong tin",
}

def inject_hard_negative(row, major, rng):
    if rng.random() >= HARD_NEGATIVE_RATE:
        return row
    
    confuse_major = CONFUSION_PAIRS.get(major)
    if not confuse_major:
        return row
    
    # Trộn thông tin từ ngành confusing vào nhưng giữ label gốc
    confuse = PROFILE[confuse_major]
    row["mon_hoc_yeu_thich"] = rng.choice(confuse["mons"])
    row["ky_nang_noi_bat"] = rng.choice(confuse["skills"])
    return row
```

```python
missing_pct = df.isnull().sum() / len(df) * 100
```


## 3.2.7 CHIA DỮ LIỆU TRAIN/TEST (TRAIN/TEST SPLIT & STRATIFIED K-FOLD)

### 3.2.7.1 Tại sao cần chia train/test?

- **Train set**: Dùng để huấn luyện mô hình
- **Test set**: Dùng để đánh giá mô hình độc lập (không từng thấy trước)
- **Holdout set**: Tập riêng biệt, hoàn toàn độc lập để kiểm tra generalization

**Lưu ý:** PHẢI chia train/test TRƯỚC làm sạch dữ liệu để tránh data leakage!

### 3.2.7.2 Stratified K-Fold trong project

**File: train_model.py (dòng 58-82)**

```python
from sklearn.model_selection import StratifiedKFold, train_test_split

def cross_validate_macro_f1(x, y, estimator, n_splits=3):
    # Stratified: giữ tỷ lệ lớp trong mỗi fold
    skf = StratifiedKFold(n_splits=n_splits, shuffle=True, random_state=42)
    
    scores = []
    for train_idx, val_idx in skf.split(x, y):
        x_tr = x[train_idx]
        x_va = x[val_idx]
        y_tr = y.iloc[train_idx]
        y_va = y.iloc[val_idx]
        
        # Train model trên fold này
        m = clone(estimator)
        m.fit(x_tr, y_tr)
        
        # Đánh giá trên validation fold
        pred = m.predict(x_va)
        macro_f1 = f1_score(y_va, pred, average="macro", zero_division=0)
        scores.append(macro_f1)
    
    # Trả về mean và std của macro_f1 qua các fold
    return mean(scores), std(scores)
```

**Ưu điểm Stratified K-Fold:**
- Giữ tỷ lệ lớp trong mỗi fold
- Tránh overfitting trên một fold
- Đánh giá model chính xác hơn

### 3.2.7.5 Best practices

1. **Giữ random_state cố định** để có thể tái hiện kết quả
2. **Dùng stratified split** cho dữ liệu không cân bằng
3. **Không chuẩn hóa scaler trên test set**
4. **Fit scaler trên train**, áp dụng lên test
5. **Kiểm tra holdout set độc lập** để đánh giá thực tế
