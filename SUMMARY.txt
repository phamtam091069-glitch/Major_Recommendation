📋 VẤN ĐỀ CHÍNH
================================================================================
Code fallback API của botchat có 4 lỗi lớn:

1. ❌ KHÔNG CÓ RETRY LOGIC
   - Gọi API 1 lần, nếu timeout/fail → Error ngay
   - Không thử lại hoặc chờ

2. ❌ ERROR HANDLING YẾU
   - Timeout không được phân biệt từ lỗi khác
   - Connection error cũng bị treat như generic error

3. ❌ KHÔNG CÓ DUAL FALLBACK
   - Chỉ dùng Claude, không có API dự phòng
   - Nếu Claude fail → Cả hệ thống fail

4. ❌ RESPONSE FORMAT KHÔNG NHẤT QUÁN
   - API trả về format khác nhau
   - Frontend khó xử lý


✅ GIẢI PHÁP
================================================================================
Tôi đã tạo file FALLBACK_API_IMPROVED.py với:

1. ✅ RETRY LOGIC (Exponential Backoff)
   - Thử 3 lần mỗi API
   - Delay: 1s, 2s, 4s (exponential)
   - Fail thế mới chuyển sang API khác

2. ✅ BETTER ERROR HANDLING
   - Phân biệt: Timeout vs HTTP error vs Network error
   - Mỗi loại lỗi có xử lý riêng
   - Chi tiết logging cho debugging

3. ✅ DUAL API FALLBACK
   - Try Claude first (3 lần)
   - Nếu fail → Try Grok (3 lần)
   - Nếu cả 2 fail → Generic response (không crash)

4. ✅ CONSISTENT RESPONSE
   - Format response nhất quán
   - Luôn có "success", "response", "source" fields
   - JSON parsing built-in

📊 SO SÁNH CŨ VS MỚI
================================================================================

SCENARIO 1: Claude API Timeout
  CŨ: 30s timeout → Error → User sees nothing
  MỚI: Retry 3 times with delay → Fallback to Grok → User sees Grok response

SCENARIO 2: Both APIs Fail
  CŨ: Error/Crash
  MỚI: Generic response (không crash)

SCENARIO 3: Network Issue
  CŨ: Lỗi
  MỚI: Retry 3 times → Fallback → Still works

SCENARIO 4: Normal Success
  CŨ: ~2s
  MỚI: ~2s (same)

SCENARIO 5: Cache Hit
  CŨ: ~1ms
  MỚI: ~1ms (same)
